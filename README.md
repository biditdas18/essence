# Essence: A Personal Embedding Cluster Framework
## for Depth-Optimized Recommendations

**The primary contribution of Essence is recovering niche 
content that matches a user's idiosyncratic taste, 
independent of popularity metrics.**

### What Essence Does

Essence operates exclusively within a single user's 
engagement history. No cross-user data is consulted at 
any stage. A user's item history is embedded using a 
pre-trained sentence encoder (all-MiniLM-L6-v2) and 
clustered into K=3 semantic centroids via K-means. 
Recommendations are generated by selecting the active 
centroid aligned with the user's recent context and 
retrieving semantically similar unseen items by cosine 
similarity.

The peer-isolation property — no cross-user interaction 
graph consulted at any stage — is not merely a privacy 
consideration. It is a depth requirement. Crowd signals 
approximate preferences at the genre level. Peer-isolation 
enables modelling at the latent feature level: the specific 
combination of characteristics a user responds to, 
independent of how other users categorise those 
characteristics. Discovery becomes mathematical, 
not accidental.

### How Essence Differs from MIND, ComiRec, and CF

| Property | CF | MIND | ComiRec | **Essence** |
|---|---|---|---|---|
| Cross-user data required | Yes | Yes | Yes | **No** |
| Multiple interest profiles | No | Yes | Yes | **Yes** |
| Model training required | Yes | Yes | Yes | **No** |
| Long-tail item recovery | Poor | Poor | Poor | **Yes** |
| Privacy-safe deployment | No | No | No | **Yes** |
| Optimised for popular items | Yes | Yes | Yes | **No** |
| **Optimised for niche items** | No | No | No | **Yes** |
| Embedding source | Co-occurrence | Co-occurrence | Co-occurrence | **Item metadata** |

**Essence is not competing with MIND or ComiRec on their 
own terms. It is solving a different problem: surfacing 
niche content that matches idiosyncratic individual taste, 
using only that user's own history, with no popularity 
signal involved.**

### Experimental Results

All systems evaluated under full-catalog ranking protocol
(no negative sampling). Absolute recall values are low
by construction — the meaningful signal is the relative
pattern across systems evaluated under identical
conditions. Lift = Recall@10 divided by expected random
recall (10 slots / catalog size).

---

**Table 1 — Last.fm-1K (Music)**
99 users · 22,767 tracks · Random baseline Recall@10 ≈ 0.0004
Embeddings: item metadata (track name + artist name)

| System | Recall@10 | Lift vs Random | LT-Recall@10 | LT / Content |
|---|---|---|---|---|
| Random | 0.0004 | 1x | 0.0001 | — |
| CF (Popularity) | 0.0081 | 20x | 0.0000 | — |
| Content (Avg Embedding) | 0.0414 | 104x | 0.0032 | 1.0x |
| **Essence (K=3)** | **0.0616** | **154x** | **0.0146** | **4.5x** |

CF achieves zero long-tail recall. Essence achieves 4.5x
above the content baseline on LT-Recall and 154x above
random overall. LT/Content = how many times better
Essence recovers niche items compared to the content
baseline (the reference at 1.0x).

---

**Table 2 — Amazon Books, Pass 1 (E-commerce, Metadata Embeddings)**
2,000 users · 61,727 items · Random baseline Recall@10 ≈ 0.0002
Embeddings: item metadata (title + author name)
Same embedding approach as Table 1 — cross-domain replication.

| System | Recall@10 | Lift vs Random | LT-Recall@10 | LT / Content |
|---|---|---|---|---|
| Random | 0.0002 | 1x | 0.0001 | — |
| CF (Popularity) | 0.0034 | 17x | 0.0000 | — |
| Content (Avg Embedding) | 0.0239 | 120x | 0.0198 | 1.0x |
| **Essence (K=3)** | **0.0280** | **140x** | **0.0254** | **1.28x** |

Pattern replicates across domains. CF scores zero
long-tail recall again. Essence achieves 140x above
random. The LT/Content ratio (1.28x) is lower than
Last.fm (4.5x) — consistent with Amazon's denser
interaction structure producing more coherent average
embeddings.

---

**Table 3 — Amazon Books, Pass 2 (E-commerce, User-Review Embeddings)**
2,000 users · 61,727 items · Random baseline Recall@10 ≈ 0.0002
Embeddings: user's own review text for consumed items (100% coverage).
Taste profiles built from first-person user language.
Candidates scored using metadata embeddings (Pass 1).

| System | Recall@10 | Lift vs Random | LT-Recall@10 | LT / Content |
|---|---|---|---|---|
| Random | 0.0002 | 1x | 0.0001 | — |
| CF (Popularity) | 0.0034 | 17x | 0.0000 | — |
| Content (Avg Embedding) | 0.0039 | 20x | 0.0038 | 1.0x |
| **Essence (K=3)** | **0.0047** | **24x** | **0.0052** | **1.36x** |

Absolute recall degrades versus Pass 1 due to two
factors: (1) embedding space mismatch — taste profiles
encoded in review-text space, candidates scored in
metadata space; (2) input noise — user reviews
sometimes describe delivery, packaging, or personal
circumstances rather than the item's content. When
noise dominates the embedding input, cluster quality
degrades.

Key finding: Essence's LT/Content ratio improves from
Pass 1 to Pass 2 (1.28x → 1.36x), suggesting the
K-means clustering mechanism remains robust to
embedding noise. The degradation is a property of
cross-space retrieval and input quality, not of the
clustering mechanism itself.

Critical design principle: what goes into the embedding
determines what Essence can find. Embedding input
curation is essential for production deployments.

### Reproducing the Results

#### Requirements
- Python 3.11+
- MacOS, Linux, or Windows
- ~4GB disk space (for Amazon data download)
- ~30 minutes runtime (M4 Pro or equivalent)

#### Install dependencies
```bash
pip install sentence-transformers scikit-learn pandas numpy requests tqdm
```

#### Last.fm-1K Experiment (Music)

```bash
# 1. Download and preprocess
python data/preprocess.py

# 2. Generate embeddings
python embeddings/generate_embeddings.py

# 3. Run all three systems and evaluate
python evaluation/evaluate.py
```

Results will print to console and match Table 1 in the
paper (99 users, 22,767 tracks).

#### Amazon Books Experiment (E-commerce)

```bash
# 1. Download raw data (~26 min, ~7GB total)
python experiments/amazon_books/step0_download.py

# 2. Preprocess and subsample to 2,000 users
python experiments/amazon_books/preprocess_amazon.py

# 3. Generate embeddings (Pass 1 metadata + Pass 2 reviews)
python experiments/amazon_books/generate_embeddings_amazon.py

# 4. Run all systems and evaluate
python experiments/amazon_books/evaluate_amazon.py
```

Results will print to console and match Tables 2 and 3
in the paper (2,000 users, 61,727 items).

#### Expected output (Amazon Books, Pass 1)

```
System            Recall@10  LT-Recall@10  LT/Content
Random            0.0002     0.0001        —
CF (Popularity)   0.0034     0.0000        —
Content (Avg Emb) 0.0239     0.0198        1.0x
Essence (K=3)     0.0280     0.0254        1.28x
```

#### Notes
- All random seeds are fixed (seed=42) — results are
  fully deterministic
- Amazon raw data is not committed to the repo due to
  size. step0_download.py downloads it automatically.
- Embedding .pkl files are excluded from the repo
  (.gitignore). They are regenerated by
  generate_embeddings.py.
- Runtime on Apple M4 Pro: ~3 min (Last.fm), ~15 min
  (Amazon embeddings), ~4 min (Amazon evaluation)
