# Essence: A Personal Embedding Cluster Framework
## for Depth-Optimized Recommendations

**The primary contribution of Essence is recovering niche
content that matches a user's idiosyncratic taste,
independent of popularity metrics.**

### What Essence Does

Essence operates exclusively within a single user's
engagement history. No cross-user data is consulted at
any stage. A user's item history is embedded using a
pre-trained sentence encoder (all-MiniLM-L6-v2) and
clustered into K=3 semantic centroids via K-means.
Recommendations are generated by selecting the active
centroid aligned with the user's recent context and
retrieving semantically similar unseen items by cosine
similarity.

The peer-isolation property (no cross-user interaction
graph consulted at any stage) is not merely a privacy
consideration. It is a depth requirement. Crowd signals
approximate preferences at the genre level. Peer-isolation
enables modelling at the latent feature level: the specific
combination of characteristics a user responds to,
independent of how other users categorise those
characteristics. Discovery becomes mathematical,
not accidental. We describe the resulting objective as
depth-optimized: the goal is to retrieve items deep in a
user's long-tail interest space, rather than to maximise
aggregate engagement on broadly popular items.

### How Essence Differs from MIND, ComiRec, and CF

| Property | CF | MIND | ComiRec | **Essence** |
|---|---|---|---|---|
| Cross-user data required | Yes | Yes | Yes | **No** |
| Multiple interest profiles | No | Yes | Yes | **Yes** |
| Model training required | Yes | Yes | Yes | **No** |
| Long-tail item recovery | Poor | Poor | Poor | **Yes** |
| No cross-user data required | No | No | No | **Yes** |
| Optimised for popular items | Yes | Yes | Yes | **No** |
| **Optimised for niche items** | No | No | No | **Yes** |
| Embedding source | Co-occurrence | Co-occurrence | Co-occurrence | **Item metadata** |

MIND (Li et al., CIKM 2019) and ComiRec (Cen et al., RecSys 2020) share
Essence's multi-interest intuition but require population-level interaction
graphs for training. Their entries above reflect architectural expectations;
they are not empirically evaluated in this work. A direct empirical
comparison is planned as future work.

**Essence is not competing with MIND or ComiRec on their
own terms. It is solving a different problem: surfacing
niche content that matches idiosyncratic individual taste,
using only that user's own history, with no popularity
signal involved.**

### Experimental Results

All systems evaluated under full-catalog ranking protocol
(no negative sampling). Recall@K is computed as
`|hits ∩ test| / |test|` (standard definition).
Absolute recall values are low by construction — the
meaningful signal is the relative pattern across systems
under identical conditions. LT-Recall = same metric
restricted to singleton train items (interacted with by
exactly 1 user in the training split); users with no
singleton test items are excluded from the LT-Recall
average.

---

**Table 1 — Last.fm-1K (Music)**
99 users · 22,767 catalog items · 16,761 singleton items (73.6%)
Chronological 80/20 split · standard Recall@K
Embeddings: item metadata (track name + artist name)

| System | Recall@10 | LT-Recall@10 | LT / Content |
|---|---|---|---|
| Random | 0.0001 | 0.0000 | — |
| Popularity | 0.0023 | 0.0000 | — |
| CF (ItemKNN) | 0.0018 | 0.0000 | — |
| Content (Avg Emb) | 0.0038 | 0.0029 | 1.00x |
| **Essence (K=3)** | **0.0119** | **0.0059** | **2.01x** |

Popularity and CF (ItemKNN) are separated baselines:
Popularity is non-personalized frequency ranking;
CF (ItemKNN) uses cosine similarity over the user-item
co-occurrence matrix (Sarwar et al., WWW 2001).
Popularity achieves zero long-tail recall by construction.
CF (ItemKNN) also achieves zero long-tail recall on this
sparse dataset: co-occurrence signal is insufficient to
surface singleton items when only 99 users are present,
and under deterministic tie-breaking CF recovers none.
Essence achieves 2.01x the long-tail recall of the
content baseline.

---

**Table 2 — Amazon Books (E-commerce, Metadata Embeddings)**
2,000 users · 61,727 catalog items · 42,878 singleton items (69.5%)
Chronological 80/20 split · standard Recall@K
Embeddings: item metadata (title + author name)
Cross-domain replication of Table 1.

| System | Recall@10 | LT-Recall@10 | LT / Content |
|---|---|---|---|
| Random | 0.0001 | 0.0000 | — |
| Popularity | 0.0021 | 0.0000 | — |
| CF (ItemKNN) | 0.0031 | 0.0137 | 1.11x |
| Content (Avg Emb) | 0.0173 | 0.0124 | 1.00x |
| **Essence (K=3)** | **0.0221** | **0.0197** | **1.59x** |

The pattern replicates. Popularity achieves zero long-tail
recall again. CF (ItemKNN) recovers marginal long-tail recall
here (0.0137, 1.11x Content): with 2,000 users, co-occurrence
signal becomes sufficient to surface some low-interaction items.
CF's long-tail recovery is thus density-dependent — zero on the
sparse Last.fm-1K (matching Popularity), marginal on the denser
Amazon Books. Essence leads on long-tail recall regardless of
dataset density (2.01x on Last.fm-1K, 1.59x on Amazon Books)
and also leads on overall Recall@10 among personalized methods.

---

**Active-cluster-selection ablation (Last.fm-1K):**
Replacing recency-based centroid selection (mean of last 10 train
items chronologically) with mean-of-all-train selection collapses
Essence toward the Content baseline with no differentiation
benefit (LT-Recall 0.0005 vs 0.0059 for recency). The recency
mechanism, not the clustering step alone, is responsible for
the long-tail advantage. Ablation verified on Last.fm-1K only.

**User-review embedding experiment (Amazon Books, Pass 2):**
As a secondary experiment, taste profiles were built from
user-authored review text rather than item metadata, with
candidates still scored by metadata embeddings. Absolute
recall degrades substantially (embedding-space mismatch +
review noise), but Essence's relative advantage over Content
persists: 2.83x LT-Recall ratio in Pass 2 vs 1.59x in Pass 1,
suggesting the K-means clustering mechanism extracts usable
interest structure even under degraded input.

### Changelog from original SSRN posting

Three methodological corrections were applied between the
original SSRN preprint and the current version, all in
the direction of a harder and more standard evaluation.
The core comparative finding is unchanged: Essence
consistently outperforms Content on long-tail recall
across both datasets, and Popularity achieves zero
long-tail recall in every configuration.

1. **CF baseline replaced.** The original "CF Baseline"
   was global popularity ranking, not collaborative
   filtering. It has been renamed Popularity and a real
   ItemKNN CF baseline (item-item cosine similarity over
   the user-item matrix) has been added as a separate
   system.

2. **Train/test split changed to chronological.** The
   original evaluation used random 80/20 hold-out per
   user. This was inconsistent with the active-cluster
   selection mechanism, which selects the centroid
   nearest to the user's most recent interactions.
   Both datasets now use a chronological 80/20 split
   (first 80% by timestamp = train, last 20% = test),
   making the recency mechanism internally consistent
   with the evaluation protocol.

3. **Recall@K metric corrected.** The original
   implementation used `hits / min(|test|, k)` as the
   denominator, which inflates recall when test sets
   are large (|test| >> k). The standard denominator
   `hits / |test|` is now used throughout. Absolute
   recall values are lower as a direct consequence —
   this is expected and correct.

### Reproducing the Results

All results are fully deterministic. Random baselines use
MD5-based per-user seeds (independent of PYTHONHASHSEED).
CF tie-breaking uses lexicographic stable sort (score desc,
item index asc). Re-running the evaluation scripts produces
the exact numbers reported in the paper.

#### Requirements
- Python 3.11+
- macOS, Linux, or Windows
- ~4 GB disk space for Amazon data (~10 GB temporary during download)
- ~30 minutes runtime (Apple M4 Pro or equivalent)

#### Install dependencies
```bash
pip install sentence-transformers scikit-learn scipy pandas numpy requests tqdm
```

#### Last.fm-1K Experiment (Music)

```bash
# 1. Download and preprocess (chronological split)
python data/preprocess.py

# 2. Generate item embeddings (all-MiniLM-L6-v2; ~2 min)
#    Output: embeddings/item_embeddings.pkl  (not committed — regenerated here)
python embeddings/generate_embeddings.py

# 3. Run all 5 systems and evaluate
python evaluation/evaluate.py --users 99 --K 3 --M 10
```

Results will print to console and match Table 1 in the
paper (99 users, chronological split, standard Recall@K).

#### Amazon Books Experiment (E-commerce)

```bash
# 1. Download raw data (~26 min, ~7 GB total)
python experiments/amazon_books/step0_download.py

# 2. Preprocess to 2,000 users (chronological split; seeded for reproducibility)
python experiments/amazon_books/preprocess_amazon.py

# 3. Generate embeddings (Pass 1 metadata; ~15 min)
#    Output: data/amazon_processed/embeddings_metadata.pkl  (not committed — regenerated here)
python experiments/amazon_books/generate_embeddings_amazon.py

# 4. Run all 5 systems and evaluate (Pass 1 metadata embeddings)
python experiments/amazon_books/evaluate_amazon_phase4.py
```

Results will print to console and match Table 2 in the
paper (2,000 users, chronological split, standard Recall@K).

#### Notes
- All results are deterministic: fixed seeds + stable tie-breaking.
  Re-running any evaluation script produces bit-identical results.
- Raw data is not committed to the repo (size). Download scripts
  fetch it automatically.
- Embedding `.pkl` files are excluded from the repo (.gitignore).
  They are regenerated by the `generate_embeddings*.py` scripts.
- Runtime on Apple M4 Pro: ~4 min (Last.fm eval),
  ~15 min (Amazon embeddings + eval)
